# 修改记录

## 1. 这个文件怎么用

本文件用于长期记录每轮实验为什么改、改了哪里、结果是否合理，以及下一轮 AI 编程时应该优先看哪些地方。以后每次改实验，都把新记录追加到最上方或“后续记录模板”下面，避免忘记参数来源。

建议每轮记录包含：

- 日期和目标
- 修改过的文件
- 关键参数
- 运行命令
- 主要结果
- 问题和下一步

## 3. 当前代码状态

当前主入口推荐使用：

```bash
python run_experiment.py
```

如果已有 checkpoint，可以跳过训练：

```bash
python run_experiment.py --skip-train
```

当前默认参数集中在两个位置：

| 位置 | 用途 |
|---|---|
| `run_experiment.py` 顶部 `EXPERIMENT` | 一键实验的集中参数，推荐优先改这里 |
| `configs/*.yaml` | 分步骤脚本使用的参数 |

注意：`run_experiment.py` 和 `configs/*.yaml` 目前参数大体一致，但不是同一个数据源。以后改参数时要么只使用 `run_experiment.py`，要么同步更新 YAML，避免两套入口跑出不同结果。

## 5. 历史修改记录

### 5.1 第一版实验

目标：验证温度相关非理想和分档补偿是否能拉开明显差距。

主要参数：

```yaml
ka: 0.0012
kb: 0.0008
sigma0: 0.015
lam: 0.012
rho: 0.65
temperature_points: [0, 25, 50, 75, 100]
```

问题：

- `Original` 在 `75°C` 降到约 `25%`，在 `100°C` 降到约 `10%`，过于接近随机猜测。
- `Improved` 仍能保持 `91%~92%`，从失效直接恢复到接近理想模型。
- `25°C` 附近可能因为分档中心不对齐出现 recovery ratio 异常。
- bin sweep 只看单温度点，容易被档位中心是否恰好对齐误导。

结论：可以证明补偿有效，但不适合作为论文主结果。

### 5.2 第二版实验

目标：减弱非理想强度，扩展温度到 `135°C`，让曲线更平滑。

主要参数：

```yaml
ka: 0.0006
kb: 0.00025
sigma0: 0.008
lam: 0.004
rho: 0.90
nominal_gain: 0.992
compensation_strength: 0.75
temperature_points: [0, 25, 50, 75, 85, 95, 105, 115, 125, 135]
```

结果特点：

- `Original` 高温下降变得渐进，但 `135°C` 仍降到约 `63.41%`。
- `Improved` 在高温区长期保持 `92%+`，过于接近 `Ideal=93.1%`。
- 逐块 MSE 降幅多数超过 `80%`，显得补偿过强。

结论：比第一版稳定，但仍不适合作为论文主结果。

### 5.3 第三版整理结果

目标：形成更适合论文表述的“有限恢复”结果。

整理结果见 `数据.md`。

当前最适合写论文的结论：

- `Original` 从 `25°C` 到 `135°C` 下降 `7.69` 个百分点。
- `Improved` 高温区平均提升 `4.12` 个百分点。
- 逐块 MSE 平均下降 `34.76%`。
- `4-bin` 与 `6-bin` 几乎相同，说明分档收益逐渐饱和。

注意：这组结果需要补跑并落盘到 `outputs/csv/`，作为最终论文结果证据。

## 6. 下一轮优先修改建议

优先级从高到低：

1. 重新运行 `python run_experiment.py --skip-train`，确认 `outputs/csv/` 能生成最新结果。
2. 如果没有 `outputs/checkpoints/best.pt`，先运行 `python run_experiment.py` 完整训练。
3. 把 `run_experiment.py` 与 `configs/*.yaml` 的默认分档策略统一，避免入口差异。
4. 检查 `scripts/eval_temperature.py` 默认使用 `temp_improved_4bin.yaml`，而旧文档里有些地方写 `5-bin`，需要以后统一术语。
5. 如果最终结果提升太弱，可以略微降低 `rho`，例如 `0.82 -> 0.78`。
6. 如果最终结果过于贴近 `ideal`，提高 `rho` 或降低 `compensation_strength`。
7. 如果 `original` 高温下降太弱，增加 `ka` 或 `kb`；如果下降太强，降低 `ka` 或 `kb`。
