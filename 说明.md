# 项目说明

## 1. 项目一句话说明

本项目用 `CIFAR-10 + ResNet18` 做一个系统级行为验证实验：先训练正常的图像分类模型，再在推理阶段把芯片部署中可能出现的温度相关非理想行为注入到每个残差块输出上，最后比较无补偿和温度自适应补偿后的精度、噪声鲁棒性和逐块特征误差。

它不是晶体管级仿真，也不声称等同真实芯片实测。项目定位是把“温度漂移、偏移、噪声、分档补偿”等电路现象抽象为神经网络推理中的行为扰动，用来支撑论文实验部分的趋势分析。

## 2. 要回答的研究问题

项目主要回答三个问题：

1. 温度升高后，未补偿硬件非理想是否会让 ResNet18 推理精度下降？
2. 温度自适应补偿能否缓解这种下降？
3. 这种补偿是“有限恢复”，还是过于理想地恢复到软件基线？

论文里更可信的结论应强调“高温下稳定但有限的恢复能力”，而不是把补偿写成完全消除硬件非理想。

## 3. 三种推理模式

| 模式 | 含义 | 用途 |
|---|---|---|
| `ideal` | 纯软件 ResNet18，不注入非理想 | 作为理想基线 |
| `original` | 注入标称部署误差、温度增益漂移、偏移漂移和随机噪声，不做补偿 | 表示未补偿芯片部署 |
| `improved` | 使用与 `original` 相同的基础非理想，但加入温度分档补偿和噪声缩放 | 表示加入补偿后的部署 |

当前实验的高温边界定义为 `85°C`。`T > 85°C` 视为高温区，用来观察补偿是否在高温区更有效。

## 4. 行为模型

对某个 residual block 输出 `z`，未补偿模式为：

```text
z_original = g(T, 25) * z + b(T, 25) + eps(T)
g(T, Tref) = nominal_gain * (1 + ka * (T - Tref))
b(T, Tref) = nominal_bias + kb * (T - Tref)
sigma(T) = sigma0 * (1 + lam * max(T - 25, 0))
eps(T) ~ N(0, sigma(T)^2)
```

补偿模式先根据温度所在区间得到档位中心 `Tc`，再使用有限补偿参考温度：

```text
Tref_comp = 25 + compensation_strength * (Tc - 25)
z_improved = g(T, Tref_comp) * z + b(T, Tref_comp) + eps_c(T)
eps_c(T) ~ N(0, (rho * sigma(T))^2)
```

关键含义：

- `nominal_gain`、`nominal_bias`：模拟即使在标称温度附近也存在的部署误差。
- `ka`、`kb`：模拟温度导致的增益和偏移漂移。
- `sigma0`、`lam`：模拟随机噪声及其随温度升高的增强。
- `bin_edges`：温度补偿分档边界。
- `compensation_strength`：补偿能力，不等于 1 时表示不能完全对齐档位中心。
- `rho`：补偿后的噪声缩放因子，越小表示噪声抑制越强。

## 5. 当前默认实验设置

当前代码和配置使用的主线参数如下：

```yaml
temperature_points: [0, 25, 50, 75, 85, 95, 105, 115, 125, 135]
noise_temperature: 75
block_mse_temperature: 105
bin_sweep_all_temps: true
```

`original` 默认参数：

```yaml
ka: 0.00048
kb: 0.00018
sigma0: 0.020
lam: 0.006
nominal_gain: 0.991
nominal_bias: 0.0
```

`improved` 默认参数：

```yaml
ka: 0.00048
kb: 0.00018
sigma0: 0.020
lam: 0.006
rho: 0.82
nominal_gain: 0.991
nominal_bias: 0.0
compensation_strength: 0.72
```

当前默认单脚本 `run_experiment.py` 的分档配置为：

```yaml
2-bin: [0, 85, 135]
4-bin: [0, 35, 65, 95, 135]
6-bin: [0, 20, 40, 65, 85, 110, 135]
```

## 7. 主要输出

实验会生成四类 CSV 和对应论文图：

| 输出 | 文件 | 说明 |
|---|---|---|
| 温度扫描 | `accuracy_vs_temp.csv` | `ideal/original/improved` 随温度变化的精度 |
| 噪声扫描 | `accuracy_vs_noise.csv` | 固定温度下，噪声强度变化对精度的影响 |
| 逐块 MSE | `blockwise_mse.csv` | 每个 residual block 相对 ideal 的特征误差 |
| 分档扫描 | `bin_sweep.csv` | 比较不同温度分档数量的补偿效果 |

图表默认输出到 `outputs/figures/`，包含 PNG 和 PDF 两种格式。

## 8. 结果解读边界

更合理的结果应满足：

- `ideal` 基本稳定在约 `93%`。
- `original` 随温度升高逐渐下降，尤其在 `T > 85°C` 后更明显。
- `improved` 全温区高于 `original`，但不应长期贴近 `ideal`。
- 高温区提升可以更明显，但最好体现“有限补偿”，不要出现从接近随机猜测直接恢复到 `90%+` 的极端叙事。
- `improved_mse` 应低于 `original_mse`，但 MSE 降幅过高时要警惕补偿过强。

## 9. 调参方向

如果 `original` 高温下降太弱，优先增大 `ka`、`kb` 或 `lam`。如果 `original` 崩得太快，优先降低 `ka`、`kb`、`lam`，最后才考虑减少注入层数。

如果 `improved` 过于接近 `ideal`，优先提高 `rho` 或降低 `compensation_strength`。如果 `improved` 提升不明显，优先降低 `rho` 或提高 `compensation_strength`。
